## Rules

- Do NOT mention commit hashes.
- Do NOT mention internal ticket IDs.
- Keep sentences concise.
- Group by domain.
- Preserve chronological order.
- Never create fake features.
- Treat terms enclosed in double quotes in the changelog or context as protected
  vocabulary: copy them exactly and never translate, normalize, or alter them.
- If a release contains only fixes, do not create a features section.
- Use bullet points.

## Output structure

Use only sections that fit the release.

- 🚀 New Features
- ✨ Improvements
- 🐛 Bug Fixes
- ⚙️ Technical
